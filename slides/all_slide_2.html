Git Cheatsheet

Here are some basic commands aimed to help you with contributing to Hacktoberfest:

// set a name that is identifiable for credit when review version history
git config --global user.name

// set an email address that will be associated with each history marker
git config --global user.email

// retrieve an entire repository from a hosted location via URL
git clone [repository url]
Eg: git clone https://github.com/projects/Repo-Name.git

// initialize an existing directory as a Git repository
git init

// list your branches. A * will appear next to the currently active branch
git branch

// create a new branch at the current commit
git checkout -b [new-branch-name]

// switch to another branch and check it out into your working directory
git checkout [existing-branch-name]

// show modified files in working directory, staged for your next commit
git status

// add files as they look now to your next commit (stage)
git add .

// commit your staged content as a new commit snapshot
git commit -m “Describe changes”

// transmit local branch commits to the remote repository branch
git push origin “branch-name”

// fetch and merge any commits from the tracking remote branch
git pull

How to fork a repository:

Most commonly, forks are used to either propose changes to someone else’s project or to use someone else’s project as a starting point for your own idea.

On the top-right corner of the page displaying the repository, click on the Fork option.
